# CV Creation using LLMs (Capstone_Project-CS01)

This project is an individual capstone implementation of a **terminal-based multi-LLM CV Creation system** using **local models via Ollama**.

The system:
- Accepts **unstructured user profile text** from a file.
- Optionally accepts a **job description** text file.
- Extracts **structured candidate information**.
- Extracts **job keywords and requirements**.
- Generates **ATS-friendly CV content** (general and, if possible, JD-tailored).
- Evaluates the generated CVs and provides **refinement suggestions**.
- Writes results to **runtime-generated files** (no input/output data is included in the submission).

## Setup Instructions

1. **Install Python**

   Ensure you have **Python 3.11+** installed.

2. **Install Ollama**

   - Install Ollama on macOS (Apple M1 / Apple Silicon).
   - Confirm installation with:

   ```bash
   ollama --version
   ```

3. **Pull Required Models**

   The default configuration uses two local LLMs:

   ```bash
   ollama pull gemma3:1b
   ollama pull gemma3:4b
   ```

4. **Prepare Input Files**

   Create your own input files (not included in submission), for example:

   - `data/profile.txt` / `data/profile.docx` / `data/profile.pdf` — unstructured user profile.
   - `data/jd.txt` / `data/jd.docx` / `data/jd.pdf` — optional job description.

## Execution Instructions

From inside the `Capstone_Project-CS01/Codebase` directory:

### General CV from Profile Only

```bash
python main.py --profile ./data/profile.txt --output-dir ./outputs
```

### General + Tailored CV with Job Description

```bash
python main.py \
  --profile ./data/profile.txt \
  --jd ./data/jd.txt \
  --extract-model gemma3:1b \
  --generate-model gemma3:4b \
  --output-dir ./outputs
```

Input files supported for `--profile` and `--jd`:
- `.txt`
- `.docx`
- `.pdf` (requires `pdfplumber`)

## Runtime Outputs

The system generates the following files in the specified `--output-dir` (default: `./outputs`):

- `structured_profile.json` — structured representation of the candidate profile.
- `jd_analysis.json` — structured job description analysis (only if JD provided).
- `cv_general.txt` — general ATS-friendly CV.
- `cv_general.docx` — general ATS-friendly CV (DOCX export).
- `cv_general.pdf` — general ATS-friendly CV (PDF export; best-effort via Pandoc).
- `cv_tailored.txt` — JD-tailored CV (only if JD provided and generation succeeds).
- `cv_tailored.docx` — JD-tailored CV (DOCX export; only if generated).
- `cv_tailored.pdf` — JD-tailored CV (PDF export; best-effort via Pandoc).
- `cv_evaluation.txt` — textual evaluation report and high-level suggestions.

## File Overview

- **`main.py`**  
  Orchestrates the pipeline: reads inputs, calls extraction and generation modules, evaluates outputs, and writes files.

- **`config.py`**  
  Contains default model names (`gemma3:1b`, `gemma3:4b`), output filenames, and basic configuration constants.

- **`llm_client.py`**  
  Calls the local Ollama HTTP API (`/api/generate`) with the requested model and prompt; used by all other modules that need an LLM.

- **`extractor.py`**  
  Uses the extraction model and dedicated prompts to convert unstructured profile text into a structured Python dictionary.

- **`jd_parser.py`**  
  Similar to `extractor.py` but for job descriptions (role, skills, keywords, responsibilities).

- **`cv_generator.py`**  
  Uses the stronger generation model to create a general CV and, if JD is available, a tailored CV.

- **`evaluator.py`**  
  Performs basic rule-based analysis (section completeness, keyword coverage) and optionally uses an LLM to suggest improvements.

- **`prompts.py`**  
  Contains all prompt templates as functions for profile extraction, JD parsing, CV generation, and evaluation.

- **`utils.py`**  
  Provides file I/O helpers, JSON parsing utilities, timestamp and logging helpers.

This structure is designed to be **academically clear**, **terminal-focused**, and **easy to explain** in the accompanying capstone report.
